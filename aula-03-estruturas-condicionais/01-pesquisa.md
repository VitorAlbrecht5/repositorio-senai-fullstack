## 1. O que é uma estrutura condicional em programação? Dê um exemplo do cotidiano usando "se... então... senão".
**Vitor:** É quando o programa toma uma decisão dependendo de uma condição. Por exemplo: Se chover, levo guarda-chuva. Senão, não levo.

## 2. Qual é a diferença entre usar apenas if e usar if/else?
**Vitor:** O if faz algo se for verdadeiro, já o else faz se for falso.

## 3. Quando usamos else if? Crie um exemplo com três ou mais possibilidades.
**Vitor:** Usamos quando tem mais de uma possibilidade, como por exemplo: Aluno com nota igual ou maior que 7: aprovado, aluno com nota 5 ou maior: em recuperação e aluno com nota 4 ou menor: reprovado.

## 4. O que significa uma condição retornar true ou false?
**Vitor:** true significa que deu verdadeiro, e o false que deu falso no resultado da condição.

## 5. Qual é a diferença entre == e === em JavaScript? Qual deles é mais recomendado?
**Vitor:**

== --> O valor é igual

=== --> O valor e o tipo são iguais

O mais recomendado é: ===

## 6. Para que servem os operadores &&, || e !? Explique com um exemplo para cada um.
**Vitor:** 

&& --> Ambas as condições precisam ser verdadeiras. Exemplo: Maior de idade? && Tem CNH?

|| --> Retorna true se qualquer condição der verdadeira. Exemplo: Pago em Dinheiro || Pago com Cartão

! --> Inverte o resultado. Exemplo: !estaBloqueado

## 7. O que o operador % faz? Por que ele é útil para descobrir se um número é par ou ímpar?
**Vitor:** Ele mostra o resto da divisão. Ele é util pois com: numero % 2 == 0 você consegue descobrir se é par ou impar.

## 8. O que é switch case? Em que situação ele fica melhor do que vários else if?
**Vitor:** Switch case é uma opção usada quando o programa precisa escolher entre várias opções diferentes. Ele é melhor de usar quando já tem as opções fixas.

## 9. Para que servem break e default dentro de um switch?
**Vitor:** break: Quebrar o programa, ou seja, ele para o programa. default: Quando a opção não se encaixa em nenhum dos outros casos, ele vai cair em default.

## 10. Por que é importante testar o mesmo programa com vários valores diferentes?
**Vitor:** Para garantir que o programa funcione corretamente com diferentes valores.
